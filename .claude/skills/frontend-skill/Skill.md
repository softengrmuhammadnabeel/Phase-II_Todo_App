## Instructions

### 1. Project Setup

- Initialize a Next.js 16+ project using App Router.
- Choose TypeScript or JavaScript.
- Install and configure Tailwind CSS.
- Set up global styles and environment configurations.

### 2. Pages (App Router)

- Create pages inside the `/app` directory.
- Structure pages according to project requirements.
- Use **server components** for static content and **client components** for interactive UI.
- Implement routing with file-based conventions of App Router.

### 3. Components

- Build reusable components in a dedicated `/components` folder.
- Use **props** for customization and **composition** for advanced UI.
- Keep components modular, maintainable, and easy to test.
- Apply consistent styling and spacing conventions across all components.

### 4. Layouts

- Use App Router **layouts** for consistent design across pages.
- Include common elements like headers, footers, and navigation.
- Apply **grid** and **flex utilities** for structured layouts.
- Ensure layouts are **responsive** across devices and breakpoints.

### 5. Styling

- Use Tailwind CSS for all UI styling.
- Maintain a consistent **color palette, typography, and spacing**.
- Create reusable styling patterns for buttons, cards, forms, and text elements.
- Implement **dark/light mode theming** if required.

### 6. Best Practices

- Keep components modular and reusable.
- Use **client vs server components** effectively for performance.
- Optimize assets like images, fonts, and icons.
- Document components, layouts, and styling conventions for team collaboration.
- Follow **responsive design principles** for mobile-first and cross-device support.
- Use semantic HTML and accessibility best practices.

## Deliverables

- Fully initialized **Next.js 16+ App Router project** integrated with Tailwind CSS.
- Pages with proper routing, layouts, and reusable components.
- **Styled, responsive, and documented components** ready for team use.
- Documentation for component API, layout structure, and styling approach.
